# CLAUDE.md ‚Äî Session-start checklist

Read this file at the start of every Claude Code session that involves
**building, auditing, or iterating** on repository content.

**If you are instead *querying* the repo to answer a user's question** ‚Äî
not modifying content ‚Äî read `AGENT.md` at the repo root first. AGENT.md
is the consumer-agent entry point; it documents the query protocol, the
research-artifact layer, and how to route to the right data without
invoking the build pipeline. CLAUDE.md below assumes a contributor
session.

## 1. Read the governing docs

In order:

1. `README.md` ‚Äî what this repository is
2. `meta/conventions.md` ‚Äî evidentiary discipline and structural rationale
3. `meta/schema.yaml` ‚Äî machine-readable spec (types, kinds, archetypes, required sections, vocabularies)
4. `meta/topic/research-queue.md` ‚Äî current priority build queue

Don't skip. Governance docs change between sessions.

## 4. Ask the user what to build

Do not scaffold nodes without direction from the user. The priority queue
at `meta/topic/research-queue.md` is a suggestion, not a mandate. Confirm
scope, archetype/kind selection, and primary-source availability before
starting.

**One *new* node per build session ‚Äî hard rule.** Do not scaffold a
second node until the first is fully populated, passes `validate.py`
(including quote-verification), and is committed. Batch node
*construction* caused the 2026-04-17 pilot failure that established
this rule. The rule restricts scaffolding new nodes only ‚Äî edits,
audits, fixes, rebuilds, and multi-node sweeps across already-built
nodes are not restricted.

**Source-read-first ‚Äî hard rule.** Every `‚úÖ Confirmed ‚Äî verified verbatim`
quote must rest on text you extracted from the archived source file in
this session (via `scripts/build/extract-source.py` for PDFs / direct
read for HTML/TXT). Not training knowledge. The verbatim-quote check in
`scripts/build/validate.py` mechanically verifies this ‚Äî but write the
node correctly to begin with by following `prompts/build.md` Phase I
Steps 2 and 6 (extract sources ‚Üí populate quotes verbatim from scratch
files).

## 5. Core scripts ‚Äî quick reference

`scripts/` has two contributor-facing tiers: `build/` for the
scaffold ‚Üí render ‚Üí validate pipeline (including the validators that
gate each phase), and `tools/` for standalone utilities (archive,
transcribe, manifest, diagnostics). `scripts/checks/` holds the
per-check modules dispatched by the validators; `scripts/lib/` holds
shared helpers. See `meta/conventions.md` "Inside `/scripts/`" for the
landing rules.

### Build pipeline ‚Äî `scripts/build/`

| Script | Purpose |
|---|---|
| `new.py` | Scaffold a node from template |
| `research-scaffold.py` | Scaffold an empty research artifact for a node |
| `extract-source.py` | Extract a primary source to plaintext (Phase I) |
| `build-from-research.py` | Regenerate a node from its research artifact (Phase II ‚Äî document / person / event / transcript / media / organization / location / finding / investigation). Per-type renderers live at `scripts/build/renderers/{type}.py`; `build-from-research.py` is the orchestrator. |
| `review-coverage.py` | Coverage / boundary / stub-linking / description-drift review (Phase III) |
| `validate.py` | Schema, structural, and verbatim-quote validation |
| `validate-research.py` | Research-artifact structural validation |
| `associate.py` | Regenerate `## Associated Nodes` sections from body links |
| `build-state.py` | Refresh this file's build-state block |

### Tools ‚Äî `scripts/tools/`

| Script | Purpose |
|---|---|
| `manifest.py` | Manage `sources/manifest.yaml` (add, verify-paths, verify-checksums, ‚Ä¶) |
| `archive.py` | Submit URLs to the Wayback Machine |
| `transcribe.py` | Download YouTube captions to `sources/transcripts/` |
| `normalize-locations.py` | Diagnostic for quote `source.location` refs ‚Äî surfaces deprecated `lines N-M` forms with actual line ranges and canonical-form proposals (read-only) |
| `check-vocab.py` | Pre-flight vocabulary check for prose-drift discipline ‚Äî pools an artifact's primary-source significant tokens (shares `lib/_common.py`'s prose-drift tokenizer with `validate-research.py`) and reports per-input-token presence. Contributor convenience for drafting `description` / `background` / `top_relevance` / `credibility_notes` / synthesis-content `.note` fields against source vocabulary. |
| `coverage-suggest.py` | Source-coverage audit aid ‚Äî for each primary source on an artifact, surfaces (a) substantive source paragraphs that no quote references and (b) capitalized terms that appear in the source but nowhere in the artifact. Forward-direction complement of the verbatim-quote check; flags likely under-extraction candidates. Read-only; contributor judges what's load-bearing vs. boilerplate. Useful at audit time on already-built nodes. |

Full build walk-through: paste `prompts/onboard.md` into a fresh
session, then `prompts/build.md` for the Phase I ‚Üí II ‚Üí III procedure.

Smoke tests live in `scripts/tests/`. Before adding or modifying a script, run:

```
bash scripts/tests/help-check.sh
```

Confirms every `scripts/{build,tools}/*.py --help` exits 0 with no
traceback ‚Äî catches syntax errors, import errors, and argparse
regressions.

**Recovering 404'd primary sources via Wayback** ‚Äî if an audit hits a
manifest entry with `status: pending` plus `wayback_date` set (live URL
dead, Wayback has a snapshot), use the fuzzy-timestamp pull workflow
in `meta/sources-access.md` "Wayback Machine fetch ‚Äî fuzzy-timestamp
URLs bypass anti-bot challenge". Exact-timestamp Wayback URLs trigger
an anti-bot challenge; fuzzy-timestamp URLs (`/web/{year}/{url}`)
redirect to the nearest snapshot and serve directly.

## 6. What this repository is not

- Not a place for speculation or secondary-source summaries
- Not advocacy for any conclusion
- Not a debunking resource
- Does not adjudicate between conflicting primary sources ‚Äî documents
  both, flags the disagreement, lets the reader judge

See `meta/conventions.md` for the full epistemic standard.
